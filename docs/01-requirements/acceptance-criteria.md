# Inkwell Agent 平台 · 验收标准（v1）

> 本文件把 [`requirements.md`](./requirements.md) 每条 `REQ-NNN` / `NFR-NNN` 在 UI 维度落实为可"是 / 否"判定的 `AC-NNN`。每条 AC 必须显式引用具体 UI-NNN（来自 [`ui-spec.md`](./ui-spec.md)）；与流程相关的判定引用 UF-NNN（来自 [`user-flow.md`](./user-flow.md)）。
>
> AC 编号一旦发布不可改；新增向后追加。

## 0. 编号约定

- AC 编号：`AC-NNN`，三位起步，递增。
- 每条 AC 单行可读：在 `<UI-XXX>` 页面，做 `<X 操作>`，看到 `<Y 结果>`。
- 一条 REQ 至少有一条 AC；NFR 同样必须挂 AC。

---

## 1. REQ 验收

### REQ-001 · 用户登录

- **AC-001**：在 UI-001 输入合法账号 + 密码并点击"登录"，能跳转到 UI-003，且 UI-003 默认落在 `我的` tab。
- **AC-002**：在 UI-001 用错误密码登录，看到 UI-001 顶错误条"账号或密码错误"，账号字段保留输入、密码字段清空保留焦点。
- **AC-003**：在 UI-001 用被锁账号登录，看到顶错误条"账号已被锁定，请联系系统管理员"。
- **AC-004**：登录成功 24 小时内重启客户端，能直接落 UI-003，不重定向到 UI-001；超过 24 小时则被路由到 UI-001。
- **AC-005**：在任意页（UI-003 ~ UI-009）从顶栏用户菜单点击"登出"，能跳转到 UI-001，且再次访问 UI-003 被重定向回 UI-001。
- **AC-006**：UI-001 不存在"自助注册"或"忘记密码（自助重置）"入口；仅展示"如忘记密码或需要开通账号，请联系系统管理员"提示文字（依 OQ-005 关闭）。

### REQ-002 · Agent 列表与基础管理

- **AC-007**：登录后在 UI-003 看到三档 tab：`我的` / `团队共享` / `我使用过`，顺序固定。
- **AC-008**：在 UI-003 `我的` tab 下点击"新建 Agent"，能跳到 UI-004 草稿状态。
- **AC-009**：在 UI-003 hover 自己的 Agent，能看到"编辑 / 删除 / 共享"快捷动作；hover 他人共享的 Agent，**不**显示"编辑 / 删除"，仅"开始对话"可用。
- **AC-010**：在 UI-003 删除自己的 Agent，弹"删除后不可恢复"二次确认，确认后该 Agent 从 `我的` 列表消失。
- **AC-011**：在 UI-004 切换"共享给团队"开关到开启，团队其他成员在 UI-003 `团队共享` tab 能看到该 Agent；关闭开关，他们看不到。
- **AC-012**：在 UI-003 `我使用过` tab 下，看到的 Agent 集合与该用户在 UI-005 进行过对话的 Agent 一致（具体语义判定由 H3 落实，本验收只检查"用户与 Agent 在 UI-005 有过任意对话"为充分条件）。
- **AC-013**：UI-003 列表采用卡片网格（3–4 列响应式），每张卡片至少展示：头像（未上传时为首字母圆形占位 + 哈希背景色）、名称、描述前 60 字、所有者、最近使用时间（OQ-012 / OQ-019 closed 2026-05-08）。

### REQ-003 · Agent 基础属性

- **AC-014**：在 UI-004 基础信息区段输入名称为空或长度 > 50，点"保存"被前后端各拒一次（字段下红字 + 后端 422）。
- **AC-015**：在 UI-004 上传头像后保存，再次进入 UI-004 头像能正确回显；UI-003 列表项也使用该头像。
- **AC-016**：在 UI-004 描述输入 > 500 字，点"保存"被拒，字段下红字校验失败。

### REQ-004 · Instructions / System Prompt

- **AC-017**：在 UI-004 Instructions 区段输入多行文本并保存，再次进入回显一致。
- **AC-018**：在 UI-005 与该 Agent 对话后进入 UI-007 调试视图，能在 trace 详情看到该 Instructions 出现在"实际拼装的 prompt"中。
- **AC-019**：在 UI-004 Instructions 输入超过 32K 字符，**保存按钮**显示警告"Instructions 较长，可能挤占模型上下文"，但保存不被阻断。

### REQ-005 · 模型选择

- **AC-020**：在 UI-004 模型与参数区段下拉模型列表，至少包含一项 Azure OpenAI（具体 SKU 由后端配置决定）。
- **AC-021**：UI-004 模型下拉的所有选项与后端配置文件声明的模型集合一致；客户端代码中**不**硬编码任何模型名（可由 H4 通过"切换后端配置后客户端列表随之变化"用例验证）。
- **AC-022**：模型不可用时，UI-004 模型下拉项灰显，hover 显示"该模型暂不可用，请联系管理员"。

### REQ-006 · 模型参数配置

- **AC-023**：在 UI-004 模型与参数区段调整 temperature / top_p / max_tokens 后保存，进入 UI-007 trace 详情，能看到这些参数出现在对应模型调用的入参中。
- **AC-024**：UI-004 模型与参数任一字段切到"使用默认"，UI-007 trace 详情中该参数项不出现或显式标记"default"。

### REQ-007 · 工具调用 / Function Calling

- **AC-025**：在 UI-004 工具区段勾选某工具并填好参数后保存。
- **AC-026**：在 UI-005 触发该工具的对话，UI-007 trace 详情能看到工具调用入参与返回值。
- **AC-027**：工具失败时，UI-005 的工具气泡内联红色标识 + 失败原因；UI-007 trace 中该步骤标"failed"并保留原因（EX-003）。
- **AC-028**：UI-004 勾选工具但缺必填参数，点"保存"被拒，字段下红字"工具 <名称> 缺少必填参数：<字段>"。

### REQ-008 · Agent Skills（agentskills.io 格式）

- **AC-029**：在 UI-004 Skills 区段上传含 `scripts/` 的 Skill 文件夹 / zip，前端**前置**拒收，弹错"v1 不支持携带 scripts/ 的 Skill"，文件不进入 Skill 库。
- **AC-030**：上传仅含 `SKILL.md` + `references/` + `assets/` 的 Skill，能成功入库并出现在 Skills 列表。
- **AC-031**：把 Skill 挂到 Agent 后在 UI-005 进行相关对话，UI-007 trace 中能看到 Discovery / Activation / Execution 三阶段命中状态。
- **AC-032**：已挂载的 Skill 在对话期被后端判定缺失（EX-008），UI-005 出现淡黄系统提示气泡；对话不阻断，UI-007 trace 标"skill load warning"。

### REQ-009 · 知识库 / RAG

- **AC-033**：在 UI-004 知识库区段上传 PDF / Word / Markdown / 纯文本，列表显示解析状态（待解析 / 解析中 / 解析成功 / 解析失败）。
- **AC-034**：在 UI-005 进行与已上传文档相关的提问，UI-007 trace 中能看到从知识库召回的具体片段。
- **AC-035**：上传不在白名单类型的文件（具体白名单 H3 定）被前置拒收，文案"<文件名> 不支持：<原因>"。

### REQ-010 · 多轮对话与长期记忆

- **AC-036**：在 UI-005 同一会话内连续 3 条以上消息，Agent 能引用之前的内容（多轮上下文连续）。
- **AC-037**：在 UI-004 长期记忆区段切换 `关` / `开（保留全文）` / `开（摘要式）`，再到 UI-005 对话，UI-007 trace 中能看到记忆片段是否被注入与注入形式（v1 仅检查模式切换在 trace 中可见）。

### REQ-011 · 触发器

- **AC-038**：在 UI-006 触发器弹窗配置 cron 表达式 `0 9 * * *` + 时区 + 启用开关，到点能看到对应编排自动执行（执行历史新增一行）。
- **AC-039**：在 UI-006 触发器弹窗选 webhook，保存后**一次性**显示完整入站 URL + Secret；用户必须勾选"已妥善保存"才能关闭弹窗；关闭后 Secret 不可再次查看。
- **AC-040**：cron 表达式格式错误，弹窗内"保存"按钮被阻断，字段下红字"cron 表达式格式错误，示例：`0 9 * * *`"。

### REQ-012 · 多 Agent 协作 / 编排

- **AC-041**：在 UI-006 可视化 DAG 画布上（OQ-013 closed 2026-05-08）创建至少包含 2 个节点的 DAG，拖拽连边形成上下游关系；点"保存"通过校验生成版本号。
- **AC-042**：在 UI-006 节点属性中将"绑定 Agent"锁到具体版本号 `v<N>`；之后该 Agent 升级到 `v<N+1>`，编排再次手动运行时仍使用 `v<N>`（UI-007 trace 中可见）。
- **AC-043**：在 UI-006 画布上形成循环（A → B → A）保存被拒，顶部错误条"编排存在循环：A → B → A，请修复"；画布内 A → B 与 B → A 两条边高亮为错误状态。
- **AC-044**：编排执行超过最大步数 / 时长（具体值 H3 定）→ 后端强制终止 → UI-006 顶部错误条"已超过最大执行步数 / 时长，已强制终止"。

### REQ-013 · 公开 API / Webhook 暴露

- **AC-045**：在 UI-004 公开 API 区段点"生成新 Token"，先弹二次确认"生成新 Token 将立即作废当前 Token，外部调用方需要重新配置。确认？"。
- **AC-046**：确认后弹 Token 一次性显示弹层：完整 Token 可见 + [复制] 按钮 + 底部勾选"我已妥善保存此 Token"；**未勾选时点击右上 × / Esc / 关闭手势均被阻断，"完成"按钮为 disabled**（OQ-016 closed 2026-05-08）。
- **AC-047**：勾选后"完成"按钮启用，点击关闭弹层后页面立即刷为掩码态（前 8 位 + 后 4 位），不再可看完整 Token。
- **AC-048**：使用旧 Token 调用公开 API（外部测试），后端返回 401（EX-005）。
- **AC-049**：使用新 Token 调用公开 API（外部测试 + 合法 Agent 标识 + 输入），能成功获得 Agent 输出。
- **AC-050**：同一 Agent 同时只存在 1 个有效 Token（依 已关闭）：再次"生成新 Token"立即作废前一个，UI-004 状态条显示"已被新 Token 取代"。

### REQ-014 · 调试 / 评测

- **AC-051**：在 UI-005 工具栏点"调试"（仅 Owner 可见），能跳到 UI-007 并自动定位到当前消息所在 trace。
- **AC-052**：在 UI-007 详情区能看到每一步：实际拼装 prompt、Skills 命中、知识库召回、工具调用入参 / 出参、模型响应（含 token 用量、模型版本、参数）。本条 AC 仅验证字段完备性，不验证视觉形态（OQ-018 closed 2026-05-08，trace 主视图的表现形式推迟到原型阶段决定）。
- **AC-053**：在 UI-007 点"保存为样本"，弹窗收集名称（必填）+ 备注；保存后样本出现在评测面板列表。
- **AC-054**：样本同名时弹窗校验失败"样本名 <名称> 已存在，请改名"。
- **AC-055**：在 UI-007 样本列表点"回放"，能新生成一条 trace（不影响原 trace），并可在该新 trace 中查看回放结果。

### REQ-015 · 版本管理

- **AC-056**：在 UI-004 任一区段保存修改，UI-008 版本列表新增一行 `v<N>`（v1 起递增）。
- **AC-057**：在 UI-008 选中任一历史版本，"diff 上一版"与"diff latest"两按钮均能高亮显示差异字段（增红删绿）。
- **AC-058**：在 UI-008 点"回滚到本版"，弹二次确认 → 确认后版本列表新增 `v<N+M>`，且 latest 指向新版本。
- **AC-059**：UI-006 编排中的节点锁定到 `v<N>`，之后该 Agent 被回滚或新增版本，编排执行仍使用 `v<N>`（AC-042 同条件验证）。

### REQ-016 · 多模态输入

- **AC-060**：在 UI-005 输入区拖入一张 PNG 图片，所选模型支持视觉时能成功提交；UI-007 trace 中可见该图片被传入模型。
- **AC-061**：所选模型不支持视觉时，UI-005 上传图片被前置拒收，文案"当前模型不支持图像输入"（EX-004）；不允许提交后失败。
- **AC-062**：在 UI-005 录音 → 完成 → 显示"转写中" → 后端 Azure Speech 转写文本回填到输入框，用户可编辑后点发送（v1 不直接发送）；UI-007 trace 中可见该消息以"文本"形式注入对话。
- **AC-063**：Azure Speech 服务不可用时，UI-005 录音上传后显示错误"语音转写服务暂不可用"；用户可改用文本输入；事件不阻断会话。
- **AC-064**：在 UI-005 拖入 PDF / Word / Markdown / 纯文本，能进入对话上下文（短文档拼接 / 长文档走 RAG，由 H3 决定路径）；UI-007 trace 中能看到文档片段被使用。

### REQ-017 · Admin 最小管理员页

- **AC-065**：仅 `is_super=true` 的 Member 在顶栏用户菜单看到"管理"入口；普通 Member 不可见。
- **AC-066**：普通 Member 直接访问 UI-009 路由，被重定向（或显示"无权限"占位 + "返回 Agent 库"按钮）。
- **AC-067**：Admin 在 UI-009 `账号` tab 对一个被锁账号点"解封"，二次确认后该账号状态变"正常"。
- **AC-068**：Admin 在 UI-009 `共享` tab 对他人共享中的 Agent 点"撤销共享"，二次确认后该 Agent 从其他成员的 `团队共享` tab 消失；Owner 原件不被删除；Agent Owner 进入 UI-004 后顶部出现状态条"该共享已被管理员撤销"。
- **AC-070**：Admin **不**能在 UI-004 编辑他人 Agent 的任意配置字段（保存按钮不出现）。

## 2. NFR 验收

### NFR-001 · 联网要求

- **AC-071**：客户端启动时主动断网，UI-001 能展示登录页表单但"登录"按钮置灰，顶部全局错误条提示"网络异常，已断开"（EX-001）。
- **AC-072**：登录后断网，任何页面（UI-003 ~ UI-009）的写操作（保存 / 发送 / 上传）按钮置灰或返回 EX-001；客户端**不**降级到本地缓存对话。

### NFR-002 · 客户端跨平台

- **AC-073**：同一份代码能在 **Windows 11**（≥ 22H2）与 **macOS 12+（Apple Silicon）** 构建并启动到 UI-001。
- **AC-074**：在 Windows 11 与 macOS 12+ 上完成 UF-001 / UF-003 / UF-004 / UF-005 后，对比两端：页面布局一致、所有按钮可点击、所有输入可写入、流式回复可看到（功能等价；视觉细节差异不算回归）。
- **AC-075**：Windows 10 / Intel Mac / macOS 11 及以下**不**列入测试矩阵；构建产物可不针对这些平台发布（依 OQ-009 关闭）。

### NFR-003 · 客户端自动锁定

- **AC-076**：登录后让客户端连续 5 分钟无键鼠输入，整页跳到 UI-002，上一页内容被全屏遮罩。
- **AC-077**：登录后让客户端主窗口失焦累计 5 分钟，整页跳到 UI-002（与 AC-076 等价）。
- **AC-078**：在 UI-002 输入正确密码点"解锁"，能恢复上一页（含滚动位置 + 未提交草稿）。
- **AC-079**：在 UI-002 锁定期间，**锁定触发之后新发起的写操作**（包括已"待重发"的消息）被拒；但**锁定触发前已开始的在途任务**（录音上传 / 文件上传 / Agent 流式回复）作为 OQ-017 closed 2026-05-08 的特例保留至完成或失败，结果在锁屏背后累积；解锁后用户可手动触发重发。
- **AC-080**：在 UI-002 多次输错密码后，账号被临时锁定（具体阈值 H3）；用户需走 UF-012 由 Admin 解封。

### NFR-005 · 对话历史持久化

- **AC-084**：用户 X 在设备 A 与 Agent Y 进行 5 条消息的对话；切到设备 B 登录同一账号、进入 UI-005 与 Agent Y，能完整看到 5 条消息（顺序 / 时间一致）。
- **AC-085**：客户端清空本地缓存（卸载 / 重装）后再次登录，对话历史仍能从后端拉取并完整展示。

### NFR-006 · 数据出境合规

- **AC-086**：客户端**不**对用户输入做脱敏（PII / 敏感字段不在客户端拦截，依 OQ-001 关闭）；UI-004 / UI-005 不出现"脱敏开关"或"敏感词过滤"配置项。
- **AC-087**：所有外发到模型厂商 / Azure Speech 的请求，在 UI-007 trace 详情中以"原文"形式可见（用于复核）。
