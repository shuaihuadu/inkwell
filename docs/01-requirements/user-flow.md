# Inkwell Agent 平台 · 用户流程（v1）

> 本文件是 H1 下半段产出。每条用户流以**文本步骤列表**写出，引用 `ui-spec.md` 的 UI-NNN 与 `requirements.md` 的 REQ / NFR / EX 编号。**不**画 ASCII 流程图替代文字描述。
>
> 流编号 `UF-NNN` 一旦发布不可改；新增向后追加。

## 0. 共用前置条件

下列前置条件除 UF-001 外对所有流默认成立；流内只描述偏离：

- 客户端已启动且后端 API 可达（NFR-001）。若中途不可达，按 EX-001 进入"网络异常"分支，停止后续步骤。
- 用户已通过 UF-001 完成登录，且未触发自动锁定（NFR-003）。若途中触发锁定，按 UF-002 解锁后回原页，未提交草稿保留；锁定触发瞬间正在进行的在途任务（录音 / 上传 / 流式回复）按 OQ-017 closed 2026-05-08 的特例处理：**任务保留到完成或失败，结果在锁屏背后累积，解锁后可见**（详见 ui-spec.md §2.5 / §5.5）。

## UF-001 · 首次登录

**对应需求**：REQ-001 / NFR-001 / NFR-003

**前置**：账号已由后端运维通过 SQL / 管理脚本创建（OQ-005 closed）。

1. 用户启动客户端 → 进入 UI-001（登录页）。
2. 用户在 UI-001 输入账号、密码，点击"登录"。
3. 客户端校验非空与长度后提交后端 → UI-001 进入"提交中"状态。
4. 成功 → 跳 UI-003（Agent 库），默认落在 `我的` tab。
5. 失败 →
   - 鉴权错误：UI-001 顶错误条提示"账号或密码错误"，回到默认状态；保留账号字段，密码字段聚焦但不清空。
   - 账号已锁：UI-001 提示"账号已被锁定"；用户需联系管理员（管理员动作走 UF-012）。
   - 速率超限：UI-001 提示"登录过于频繁，请稍后重试"。
6. 网络异常：UI-001 顶部全局错误条置顶（EX-001），登录按钮置灰；自动重连成功后用户可重试。

**异常路径**：

- 用户 5 分钟内未输完密码 / 离开应用，下次回前台不会触发锁定（未登录无锁定状态）；登录页保持。
- 登录成功后 24 小时内重启客户端不需要重新登录（依 REQ-001 验收"登录态可持久 24 小时"）；超过 24 小时回 UI-001。

## UF-002 · 自动锁定与解锁

**对应需求**：NFR-003 / EX-006 / REQ-017（被锁后由 Admin 解封）

**触发**：客户端累计 5 分钟无键鼠输入或主窗口失焦。

1. 客户端在任意页面（UI-003 ~ UI-009）触发条件 → 整页跳 UI-002（锁定页），上一页内容被全屏遮罩。
2. 用户在 UI-002 输入密码，点击"解锁"。
3. 通过 → 恢复上一页（保留滚动位置 + 未提交草稿）。
4. 密码错误 → UI-002 显示"密码错误"，密码字段清空保留焦点。
5. 多次失败（具体阈值由 H3 决定）→ 后端临时锁账号，UI-002 显示"多次解锁失败，账号已临时锁定"；用户需走 UF-012 由 Admin 解封。
6. 用户点击"切换账号 / 登出" → 终止 session 跳 UI-001；草稿丢失。

**异常路径**：

- 锁定触发瞬间正在录音 / 文件上传中 / Agent 流式回复中：按 OQ-017 closed 2026-05-08 的特例处理——任务保留到完成 / 失败，结果在 UI-002 背后累积；解锁后用户回到 UI-005 看到转写文本、Agent 回复、上传解析状态。锁定期间不允许新发起任何写操作（不能触发新录音 / 新上传 / 新对话）。
- 解锁过程中网络异常：UI-002 全局错误条 + 解锁按钮置灰；恢复后用户重试。

## UF-003 · 浏览 Agent 库

**对应需求**：REQ-002 / 场景 S1 / OQ-010

1. 用户在 UI-001 登录成功 → 默认落 UI-003 `我的` tab。
2. UI-003 异步加载列表 → 加载中 → 有数据（按"最近使用时间"倒序）。
3. 用户切换 tab：`我的` ↔ `团队共享` ↔ `我使用过`，列表内容按规则切换。
   - `我的`：当前用户为 Owner 的 Agent。
   - `团队共享`：他人共享出的、当前用户可只读使用的 Agent。
   - `我使用过`：当前用户与其有过对话历史的 Agent，可与上两档重叠。
4. 用户在搜索框输入关键字 → 即时过滤当前 tab 名称包含关键字的项。
5. 用户点击列表项 → 跳 UI-004（Owner 进入编辑模式；非 Owner 进入只读模式）。
6. 用户 hover 列表项 → 显示快捷动作"开始对话"、Owner 额外"编辑 / 删除 / 共享 / 撤销共享"。
7. Admin（`is_super=true`）从顶栏用户菜单"管理"入口跳 UI-009。

**异常路径**：

- 列表加载失败：UI-003 列表区错误占位 + 重试按钮（EX-001 / EX-002）。
- 删除自己 Agent：弹"删除后不可恢复"二次确认；确认后从 `我的` 移除；其他 tab 也不再展示。
- 撤销自己共享：弹"已使用过的成员将无法继续访问"二次确认；确认后从 `团队共享` 移除（对其他用户）。
- UI-005 对话页的历史会话侧栏（OQ-014 closed 2026-05-08）主要从 UI-003 列表项 hover 动作"开始对话"进入；进入后在 UI-005 左侧侧栏查看历史会话（详见 UF-005 步骤 1）。

## UF-004 · 新建并配置 Agent

**对应需求**：REQ-002 ~ REQ-013 / 场景 S2

1. 用户在 UI-003 右上点击"新建 Agent" → 跳 UI-004（draft 状态）。
2. 用户在 UI-004 左侧区段导航依次进入 `基础信息`：
   - 输入名称（必填，1–50 字）；上传头像（可选）；描述（≤ 500 字）。
3. 进入 `Instructions`：写 System Prompt；超过 32K 字符时显示警告但不阻断。
4. 进入 `模型与参数`：从下拉选择模型（至少含 Azure OpenAI）；按需调整 temperature / top_p / max_tokens 或保留默认。
5. 进入 `工具`：勾选已注册工具，填写工具特定参数。
6. 进入 `Skills`：从 Skill 库勾选挂载；如本地有新 Skill 走 UF-006 上传分支。
7. 进入 `知识库`：上传文档（PDF / Word / Markdown / 纯文本），等待解析成功。
8. 进入 `长期记忆`：选择 `关` / `开（保留全文）` / `开（摘要式）`。
9. 进入 `触发器`：可选；详见 UF-008。
10. 进入 `共享`：可选切换"共享给团队"开关。
11. 进入 `公开 API`：可选生成 Token；详见 UF-010。
12. 用户点击顶部"保存" → UI-004 提交中 → 成功后顶部 toast"已保存为 v1"，状态从 `draft` 变 `已发布`，默认保留在当前区段（用户可在 toast 中点"查看版本"跳 UI-008）。
13. 用户点击"开始对话" → 跳 UI-005，使用刚保存的 v1 进入对话。

**异常路径**：

- 名称为空 / 超长：基础信息字段下红字校验失败，保存按钮不阻断其他字段，但点保存时整体校验不通过（详见 UI-004 §4.5）。
- 上传 Skill 含 `scripts/`：上传组件前置拒收（REQ-008 / EX-008）。
- 模型选项灰显（不可用）：用户无法选中；hover tip 显示原因；不影响保存。
- 公开 API Token 生成走单独 UF-010。
- 保存超时 / 网络异常：错误条提示"保存失败"，草稿保留，用户可重试。
- 用户离开页面（路由切换 / 关闭窗口）前有未保存改动：客户端弹"有未保存的修改，是否离开？"二次确认（v1 仅做客户端拦截，不做后台自动保存）。

## UF-005 · 与 Agent 对话（含多模态）

**对应需求**：REQ-010 / REQ-016 / 场景 S3 / EX-001 ~ EX-004 / EX-008

1. 用户在 UI-003 / UI-004 进入 UI-005。
2. UI-005 异步加载历史消息（NFR-005 多端一致）→ 显示消息流；左侧同时加载"历史会话侧栏"（OQ-014 closed 2026-05-08），侧栏顶部为"+ 新建会话"按钮，下方按时间倒序列出该 Agent 下当前用户参与的会话。用户可点击会话切换上下文，或点击"+ 新建会话"开启一个空的会话。
3. 用户在输入区操作其一：
   - **文本**：输入文字，点"发送"或按 Enter（Shift+Enter 换行）。
   - **图片**：拖入或点上传，预览缩略图；当前模型不支持视觉时上传组件前置拒收（EX-004）。
   - **语音**：点麦克风进入录音态 → 显示波形与时长 → 点"完成"上传 → "转写中" → 后端 Azure Speech 转为文本（REQ-016）→ 文本回填到输入框，用户可编辑后发送（不直接发送，给用户校对机会）。
   - **文档**：拖入或点上传，预览文件名 + 类型；解析中 → 解析成功 → 用户继续输入文字后发送。
4. 用户点"发送" → 用户消息入消息流，状态"发送中" → Agent 流式响应 → 显示"停止生成"按钮。
5. Agent 在响应中触发工具调用 → 工具气泡内联展示"调用 <工具名>，参数 <…>"；返回后继续生成。
6. 流式结束 → 消息状态变"已完成"。

**异常路径**：

- 网络异常（EX-001）：顶部全局错误条；输入区禁用；正在"发送中"的消息变"待重发"，恢复后可手动重发。
- 模型不可用（EX-002）：错误气泡，提供"重试"。
- 工具失败（EX-003）：工具气泡内联红色标识，注入失败原因到上下文；Agent 可继续推理。
- 图片不兼容（EX-004）：前置拒收。
- Skill 加载失败（EX-008）：消息流上方淡黄系统提示气泡"Skill <名称> 加载失败"，本轮回答未包含该 Skill 上下文。
- 文件超限（EX-009）：前置拒收，文案"<文件名> 不支持：<原因>"。
- 录音 / 上传 / 流式中触发自动锁定：按 OQ-017 closed 2026-05-08 的特例处理——任务保留到完成 / 失败，结果在锁屏背后累积；解锁后用户回到 UI-005 看到全部转写 / 回复 / 上传结果。锁定期间发起新录音 / 新上传 / 新对话被路由拦截到 UI-002。
- 用户中断生成：点"停止生成" → 当前已生成内容保留为最后一条消息，标"已停止"。

## UF-006 · 配置工具与 Skills

**对应需求**：REQ-007 / REQ-008 / 场景 S4 / EX-008

1. 用户在 UI-004 进入 `工具` 区段：
   1. 列表展示后端注册工具集合，每项可勾选挂载到 Agent。
   2. 勾选后该工具的特定参数表单展开；用户填写必填项。
   3. 缺少必填参数 → 字段下红字"工具 <名称> 缺少必填参数：<字段>"；阻断保存。
2. 用户在 UI-004 进入 `Skills` 区段：
   1. 上方"上传 Skill"按钮 → 选择本地文件夹 / zip。
   2. 客户端**前置**扫描结构：发现 `scripts/` 目录立即弹错"v1 不支持携带 scripts/ 的 Skill"（REQ-008）；阻断上传。
   3. 通过前置 → 上传 → 后端二次校验（`SKILL.md` 可解析）→ 入 Skill 库；列表新增项。
   4. 后端校验失败 → 上传气泡红色标识，hover 显示原因；不影响其他 Skill。
3. 用户从 Skill 列表勾选挂载到当前 Agent → 保存。

**异常路径**：

- Skill 列表加载失败：列表占位"加载失败"+ 重试。
- 上传过程网络异常：进度条出现"上传失败"标识，可重试。
- 已挂载的 Skill 在对话期被后端判定缺失 / 损坏（EX-008）：UI-005 系统提示气泡，本轮不阻断。

## UF-007 · 调试与回放对话

**对应需求**：REQ-014 / 场景 S5

1. 用户在 UI-005 工具栏点击"调试"（仅 Owner）→ 跳 UI-007，自动定位到当前消息所在 trace。
2. UI-007 左侧 trace 列表，右上详情区按层级展示：实际拼装 prompt → Skills 命中 → 知识库召回片段 → 工具调用入参 / 出参 → 模型响应（含 token 用量、模型版本、参数）。
3. 用户点详情区某一步右上"复制 prompt" → 内容入剪贴板。
4. 用户点"保存为样本" → 弹窗收集名称 + 备注 → 样本入评测面板。
5. 用户在样本上点"回放" → 客户端把样本输入再次发往后端 → 列入新的 trace（不影响原 trace）。
6. 用户点"返回该对话" → 跳回 UI-005 该消息所在位置。

**异常路径**：

- 样本同名：弹窗校验失败"样本名 <名称> 已存在，请改名"。
- 回放被速率限制：错误条 + 倒计时。
- trace 缺片段：详情区对应段落显示"<段名> 数据缺失"。
- 非 Owner 进入 UI-007：仅显示自己作为对话发起人的 trace（详见 UI-007 §7.8）。

## UF-008 · 配置编排与触发器

**对应需求**：REQ-011 / REQ-012 / 场景 S6 / EX-007

1. 用户从 UI-003 顶部入口进入 UI-006 编排列表 → "新建编排"或编辑已有编排。
2. 用户在**可视化 DAG 画布**（OQ-013 closed 2026-05-08）上添加节点：从左侧节点库拖拽或点击画布工具栏"添加节点"：
   - 节点名称（编排内唯一）。
   - 绑定 Agent 并锁定具体版本号（默认 latest，提示锁定建议）。
   - 输入映射：上游节点输出 → 当前节点输入字段，至少一条。
3. 用户连边形成 DAG。
4. 用户点"保存" → 客户端做基础校验（无未绑定 / 无循环）→ 后端二次校验。
5. 用户点"触发器"打开弹窗 → 选 cron 或 webhook：
   - cron：填表达式 + 时区；启用开关。
   - webhook：保存后**一次性显示**入站 URL + Secret；用户必须在弹窗中确认"已妥善保存"才能关闭。
6. 用户点"手动运行" → 弹窗收集入参 → 编排执行 → 节点级 loading → 完成后跳"历史"子页查看本次运行 trace（可联跳 UI-007）。

**异常路径**：

- 校验失败（循环 / 未绑定 / 缺映射 / cron 非法）：顶部错误条列出问题；阻断保存。
- 执行超出最大步数 / 时长（EX-007）：顶部错误条"已超过最大执行步数 / 时长，已强制终止"；事件入审计日志（NFR-004）。
- 触发器 webhook Secret 关闭弹窗后再次访问：仅显示掩码态，不可再查看完整 Secret。
- 在 UI-006 编辑过程中触发自动锁定：返回后保留草稿（见 UF-002 共用规则）。

## UF-009 · 版本管理与回滚

**对应需求**：REQ-015 / 场景 S7

1. 用户在 UI-004 顶部点"版本与调试" → "版本" → 跳 UI-008。
2. UI-008 左侧版本列表（v1 起递增），默认选中 latest。
3. 用户点击任一历史版本 → 详情区展示该版本字段；提供 "diff 上一版" / "diff latest"。
4. 用户点"回滚到本版" → 二次确认"回滚后将生成新版本 `v<N+1>`，引用 latest 的编排 / 公开 API 仍指向 latest"。
5. 确认 → 回滚中 → 成功 toast → 列表新增 `v<N+M>`。
6. 编排（UI-006）锁定到 `v<N>`（被回滚的版本）的节点不受新版本影响（REQ-015 验收）。

**异常路径**：

- 回滚失败：错误条提示，未生成新版本。
- 跨破坏性变更回滚：弹"该回滚会丢弃以下字段：<列表>，确认？"（具体破坏性判定 H3）。

## UF-010 · 公开 API Token 生成与轮换

**对应需求**：REQ-013 / OQ-004 / EX-005

1. 用户在 UI-004 进入 `公开 API` 区段。
2. 当前 Token 三态之一：
   - 未生成：占位"还未生成 Token" + 主按钮"生成新 Token"。
   - 已生成：显示"前 8 位 + 后 4 位"掩码 + "生成新 Token"按钮。
   - 已被新 Token 取代：仅记入审计；UI 上不再展示。
3. 用户点"生成新 Token" → **先**弹二次确认"生成新 Token 将立即作废当前 Token，外部调用方需要重新配置。确认？"。
4. 确认后 → 弹 Token 一次性显示弹层：
   - 显示完整 Token；提供[复制]按钮。
   - 底部勾选"我已妥善保存此 Token"，勾选后"完成"按钮才可用。
   - 关闭弹层 → 页面立刻刷为掩码态；完整 Token 不可再次查看。
5. 弹层下方 curl 调用样例（参数化展示 Token 与 Agent 标识）。
6. 外部系统使用新 Token 调用 → 记入审计日志（NFR-004 / REQ-017）。
7. 旧 Token 调用 → 后端返回 401（EX-005），事件入审计。

**异常路径**：

- Token 生成失败：弹层不出现；区段顶错误条"Token 生成失败：<错误码>"；旧 Token 状态保持。
- 用户未勾选"已保存"直接关闭弹层：客户端**阻断关闭**（OQ-016 closed 2026-05-08）——右上 × / Esc / 关闭手势均被拦截，提示"请勾选『我已妥善保存此 Token』才能关闭」。用户只能勾选后点击"完成"关闭弹层；Token 始终保留有效，不被隐含作废。
- 网络异常：按 EX-001 处理。

## UF-011 · Owner 共享 Agent · Admin 撤销共享

**对应需求**：REQ-002 / REQ-017 / OQ-007

### UF-011-A · Owner 共享 / 撤销自己共享

1. Owner 在 UI-003 hover 自己的 Agent → "共享" / "撤销共享"；或在 UI-004 `共享` 区段切换开关。
2. "共享给团队"开启 → 该 Agent 出现在团队成员的 `团队共享` tab；只读使用。
3. "撤销共享"或开关关闭 → 弹"已使用过的成员将无法继续访问"二次确认 → 确认后立即生效。

### UF-011-B · Admin 撤销他人共享

1. Admin（`is_super=true`）从顶栏用户菜单"管理"进入 UI-009 → 切到 `共享` tab。
2. 列表显示当前所有共享中的 Agent（Agent 名 / Owner / 共享开始时间）。
3. Admin 点击行右"撤销共享" → 二次确认"撤销 <Agent 名> 的共享后，已使用的成员将无法继续访问。确认？"。
4. 确认 → 后端撤销共享 + 入审计日志（NFR-004，事件类型 `admin_revoke_share`）。
5. 该 Agent 的 Owner 在 UI-004 顶部出现状态条"该共享已被管理员撤销"。

**异常路径**：

- 撤销共享失败：UI-009 错误条 + 列表保持原状。
- 普通 Member 路由到 UI-009：显示"无权限"占位 + "返回 Agent 库"按钮。

## UF-012 · Admin 解封账号

**对应需求**：REQ-017 / NFR-003

1. Admin 进入 UI-009 → 切到 `账号` tab。
2. 列表显示当前已锁账号；可按用户名搜索 / 状态筛选。
3. Admin 点击行右"解封" → 二次确认"将解封 <用户名>，是否继续？"。
4. 确认 → 解封中 → 成功后状态变"正常"，列表刷新；事件入审计日志。
5. 被解封用户下次登录 UI-001 → 正常通过。

**异常路径**：

- 解封失败：UI-009 错误条；列表保持原状。
- Admin 自己被锁：本流不适用（自己被锁后无法登录，需另一位 Admin 解封；若团队仅一名 Admin，按 §3 / OQ-007 由后端运维 SQL 兜底）。

## UF-013 · Admin 查询审计日志

**对应需求**：REQ-017 / NFR-004

1. Admin 进入 UI-009 → 切到 `审计日志` tab。
2. 筛选条件：用户、Agent、时间范围、事件类型（登录 / Agent CRUD / Agent 调用 / 版本回滚 / 公开 API 调用）。
3. 提交筛选 → 列表加载 → 有数据。
4. 用户点击条目 → 弹层显示完整字段（具体字段 H3 决定）。
5. v1 不提供导出（OQ-020 closed 2026-05-08，选 B）：UI-009 §9.4 不出现"导出"按钮；Admin 可对条目查看详情，合规导出场景由后端运维兜底（SQL / 后台脚本）。

**异常路径**：

- 加载超时：错误条"审计日志加载超时，请缩小筛选范围或重试"。
- 时间范围非法：客户端校验阻断；提示"开始时间需早于结束时间"。

## UF-014 · 异常路径合集

> 本节汇总跨流程异常分支的典型走法，避免在每条流里重复。

### UF-014-A · 后端不可达（EX-001）

- 全局错误条置顶；所有写操作禁用；登录页登录按钮置灰。
- 已发出但未确认的请求保留为"待重发"，恢复后用户手动触发。
- 客户端**禁止**降级到本地缓存对话。

### UF-014-B · 模型故障（EX-002）

- UI-005 错误气泡 / UI-006 顶部错误条；按错误码渲染重试建议；不静默失败。

### UF-014-C · 工具调用失败（EX-003）

- UI-005 工具气泡内联红色标识；失败原因注入对话上下文；Agent 可继续推理或终止本轮。

### UF-014-D · 多模态不兼容（EX-004）

- 上传 / 录音前置拒收；用户在输入区即收到提示，不允许提交后失败。

### UF-014-E · 公开 API 鉴权失败（EX-005）

- 仅出现在外部调用方（HTTP 4xx + 错误码）；客户端 UI 不展示；UI-009 审计日志可见。

### UF-014-F · 自动锁定（EX-006）

- 见 UF-002。

### UF-014-G · 编排死循环 / 超时（EX-007）

- 后端强制终止；UI-006 顶部错误条；事件入审计日志（包含触发 ID、终止原因、终止时间）。

### UF-014-H · Skill 加载失败（EX-008）

- UI-005 系统提示气泡（淡黄）；UI-004 Skills 区段对应卡片标"加载失败"；不阻断对话。

### UF-014-I · 文件超限 / 类型不支持（EX-009）

- 上传组件前置拒收，文案"<文件名> 不支持：<原因>"；后端二次校验作为兜底。
