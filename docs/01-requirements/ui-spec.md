## 1. UI-001 · 登录页

**用途**：未登录用户进入应用时的第一屏；不显示其他任何页面入口。

### 1.1 布局

单列居中卡片：

- 上：应用 Logo 与名称、副标题（一句话定位）
- 中：账号输入框、密码输入框（带"显示密码"切换）
- 下：主按钮"登录"；下方一行说明文字"如忘记密码或需要开通账号，请联系系统管理员"（依 OQ-005 关闭——账号由后端运维管理；不提供自助重置）

### 1.2 状态

| 状态   | 触发                   | 表现                                                                   |
| ------ | ---------------------- | ---------------------------------------------------------------------- |
| 默认   | 进入页面               | 表单为空；账号字段获得焦点；登录按钮启用                               |
| 提交中 | 点击"登录"             | 登录按钮置 loading 态、字段只读；其他交互禁用                          |
| 失败   | 后端返回鉴权错误       | 错误条出现在表单顶部，给出错误码与一句话说明；账号 / 密码字段不清空    |
| 已锁   | 后端返回账号被锁       | 错误条文案"账号已被锁定，请联系管理员"，参考 EX-005 / REQ-017 解封流程 |
| 离线   | 启动时检测到后端不可达 | 全局错误条置顶，按 EX-001 提示"网络异常，已断开"；登录按钮置灰         |

### 1.3 表单字段

| 字段 | 必填 | 客户端校验      | 后端二次校验        |
| ---- | ---- | --------------- | ------------------- |
| 账号 | 是   | 非空；长度 1–64 | 是否存在 / 是否已锁 |
| 密码 | 是   | 非空            | 是否匹配            |

### 1.4 操作按钮

| 按钮     | 行为                        |
| -------- | --------------------------- |
| 登录     | 提交；成功后跳 UI-003       |
| 显示密码 | 切换密码字段明文 / 暗文显示 |

### 1.5 错误提示

| 错误码（语义）            | 文案（v1 仅 zh-CN，OQ-015 closed 2026-05-08；不引入 i18n key 体系） |
| ------------------------- | ------------------------------------------------------------------- |
| 账号或密码错误（401）     | 账号或密码错误，请重试                                              |
| 账号已锁（423）           | 账号已被锁定，请联系系统管理员                                      |
| 速率超限（429）           | 登录过于频繁，请稍后重试                                            |
| 后端不可达（NET-OFFLINE） | 网络异常，已断开。请检查网络连接                                    |

### 1.6 空状态

不适用（登录页无列表 / 数据集合）。

### 1.7 加载状态

仅"提交中"使用按钮内置 loading 指示符；不引入全屏 spinner / 骨架屏（v1 加载与空状态采用"骨架占位 + 文案"形式，OQ-021 closed 2026-05-08，详见 §10.2 / §10.3）。

### 1.8 权限差异

仅"未登录"角色可见此页；已登录访问 `/login` 直接重定向到 UI-003（已锁定状态走 UI-002）。

### 1.9 关键交互流程

见 [`user-flow.md` UF-001](./user-flow.md#uf-001-首次登录)。

## 2. UI-002 · 锁定页

**用途**：客户端累计 5 分钟无操作或主窗口失焦后强制跳入；解锁前禁止任何对后端的写操作（NFR-003 / EX-006）。

### 2.1 布局

全屏遮罩，遮挡所有上一页内容（不展示 Agent 列表、对话内容等敏感信息）：

- 中：当前登录用户的头像 + 用户名；密码输入框；"解锁"按钮
- 下：辅助操作"切换账号 / 登出"（点击则结束当前会话并跳 UI-001）

### 2.2 状态

| 状态     | 触发                 | 表现                                         |
| -------- | -------------------- | -------------------------------------------- |
| 默认     | 触发自动锁定         | 密码字段聚焦；已上一页全部不可见             |
| 解锁中   | 点击"解锁"           | 按钮 loading；字段只读                       |
| 解锁失败 | 密码错误             | 错误条提示"密码错误"；密码字段清空但保留焦点 |
| 离线     | 解锁过程中后端不可达 | 错误条按 EX-001 提示，解锁按钮置灰           |

### 2.3 表单字段

| 字段 | 必填 | 校验                       |
| ---- | ---- | -------------------------- |
| 密码 | 是   | 非空、与服务端账号密码匹配 |

### 2.4 操作按钮

| 按钮     | 行为                                          |
| -------- | --------------------------------------------- |
| 解锁     | 通过则恢复上一页（保留滚动位置 / 未提交草稿） |
| 切换账号 | 终止当前 session 并跳 UI-001                  |
| 登出     | 终止当前 session 并跳 UI-001（不保留草稿）    |

### 2.5 错误提示

| 错误码（语义）         | 文案                                       |
| ---------------------- | ------------------------------------------ |
| 密码错误（401）        | 密码错误，请重试                           |
| 多次失败（429 / 锁定） | 多次解锁失败，账号已临时锁定。请联系管理员 |
| 后端不可达             | 网络异常，已断开。请检查网络连接           |

### 2.6 空状态

不适用。

### 2.7 加载状态

按钮内置 loading（解锁中）。

### 2.8 权限差异

仅"已登录但被锁定"状态出现；其他状态访问应被路由层拦截。

### 2.9 关键交互流程

见 [`user-flow.md` UF-002](./user-flow.md#uf-002-自动锁定与解锁)。`is_super=true` 在锁定状态下与普通 Member 无差异。

### 2.5 在途任务特例（OQ-017 closed 2026-05-08）

锁定触发瞬间存在的**在途任务**——具体指：正在录音的语音输入、文件上传中、Agent 流式回复中——按以下特例处理：

- **保留在途任务到完成或失败**：客户端不主动取消已发起的任务；后端继续执行至自然终止（成功 / 失败 / 超时）。
- **结果在锁屏背后累积**：UI-002 在前景遮挡所有内容；任务完成 / 失败的状态、转写结果、流式回复内容写入对应 UI-005 会话历史，不在锁屏期间向用户展示。
- **解锁后呈现**：用户在 UI-002 输入正确密码后回到 UI-005，看到累积期间的所有结果与状态变化（例如转写文本已注入输入框、Agent 回复已完成、上传已解析）。
- **NFR-003 "禁止写操作" 语义特例**：锁定触发**前**已发起的在途任务允许继续完成；锁定**期间**禁止新发起任何写操作（包括不允许触发新的录音、上传、对话）。

> 本特例只在 ui-spec.md / user-flow.md / acceptance-criteria.md 落账；requirements.md NFR-003 字面表述（"5 分钟无操作或失焦自动锁定"）不变。

## 3. UI-003 · Agent 空间

**用途**：登录后的主入口；列出当前用户可见的 Agent，支持新建。两档 tab（原三档，2026-07-15 移除"我使用过"，详见 [requirements.md §13 第 32 条](./requirements.md)）依 [OQ-010 closed](./open-questions.md#oq-010-v1-是否需要团队共享-agent-库独立于我的-agent)。

### 3.1 布局

- 顶：标题"Agent 空间" + 右侧"新建 Agent"主按钮
- 中：两档 tab——`我的` / `团队共享`（顺序固定）
- 下：列表区采用**卡片网格 3–4 列响应式**（OQ-012 closed 2026-05-08）；每张卡片含头像、名称、描述前 60 字、Owner、最近使用时间、共享状态徽标。卡片内头像未上传时使用首字母圆形占位、背景色按字符哈希分配（OQ-019 closed 2026-05-08）；右上提供"刷新"与"搜索"（按名称模糊搜）

### 3.2 状态

| 状态           | 表现                                                                                              |
| -------------- | ------------------------------------------------------------------------------------------------- |
| 加载中         | 列表区**卡片网格骨架占位**（行高一致的灰色矩形卡片），tab 可见可点；新建按钮可用（OQ-021 closed） |
| 空（我的）     | 占位文案"还没有自己的 Agent，点击"新建 Agent"开始"；新建按钮高亮                                  |
| 空（团队共享） | 占位文案"团队成员还没有共享 Agent"                                                                |
| 有数据         | 默认按"最近使用时间倒序"排序（v1 不提供排序切换；具体排序键由 H3 决定）                           |
| 出错           | 列表区错误占位"加载失败，[重试]"，按 EX-001 / EX-002 提示；新建按钮仍可用                         |

### 3.3 表单字段

仅顶部"搜索"输入框：模糊匹配名称（不命中描述 / 所有者）；最长 50 字符；不必填。

### 3.4 操作按钮

| 按钮              | 来源         | 行为                                                                                                                                                                                                |
| ----------------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 新建 Agent        | 页面右上     | 直接进入 UI-004（draft 状态空白配置）                                                                                                                                                               |
| 刷新              | 列表右上     | 重新拉列表                                                                                                                                                                                          |
| 搜索              | 列表右上     | 即时筛选当前 tab 的 Agent                                                                                                                                                                           |
| 列表项点击        | 列表项整体   | 已发布 Agent（含“有未发布的修改”状态）→ 直达 UI-005 对话页，使用当前已发布版本；仅存在未发布过的草稿 → 跳 UI-004 详情/编辑页（非 Owner 只读）（2026-07-15 修订，详见 requirements.md §13 第 30 条） |
| 列表项 hover 操作 | 列表项右侧   | Owner：编辑（跳 UI-004）/ 删除 / 共享 / 撤销共享；非 Owner 无 hover 操作（2026-07-15 修订，原“开始对话”改由列表项点击本身承担）                                                                     |
| Admin 入口        | 顶栏用户菜单 | 仅 `is_super=true` 可见，跳 UI-009                                                                                                                                                                  |

### 3.5 错误提示

| 场景           | 文案                                                           |
| -------------- | -------------------------------------------------------------- |
| 删除自己 Agent | 二次确认弹层"删除后不可恢复，确认删除？"                       |
| 撤销自己共享   | 提示"撤销共享后，已使用过的成员将无法继续访问该 Agent。确认？" |

### 3.6 空状态

见 3.2 表"空（我的 / 团队共享）"两档独立空态文案。

### 3.7 加载状态

## 4. UI-004 · Agent 配置 / 详情页

**用途**：单个 Agent 的全部配置与元信息查看。覆盖 REQ-003 ~ REQ-013 全部配置维度。

### 4.1 布局

页面分两栏（外壳沿用 §0.2 "顶栏 + 左侧 nav + 主区"，OQ-011 closed）：

- 左侧（区段导航）：列出全部 10 个区段：`基础信息` / `Instructions` / `模型与参数` / `工具` / `Skills` / `知识库` / `长期记忆` / `共享` / `公开 API` / `版本与调试`
- 右侧（区段内容）：渲染当前选中区段；顶部固定"开始对话"主按钮（跳 UI-005）、"存为草稿"按钮与"发布"按钮；标题区显示 Agent 头像、名称、当前版本号 / 发布状态（draft / 已发布），若存在尚未发布的草稿改动额外显示"有未发布的修改"标签、Owner

### 4.2 状态

| 状态           | 表现                                                                                                                              |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| 新建草稿       | URL 标识 `new`；顶部状态条"未发布的草稿"；发布后自动落到 v1 并跳具体 ID                                                           |
| 编辑           | 任意区段字段修改即在该区段标"未保存"小点；离开页面前提示                                                                          |
| 有未发布的修改 | 该 Agent 已存在已发布版本，但当前编辑内容已"存为草稿"且尚未发布；标题区显示"有未发布的修改"标签；"开始对话"仍使用上一个已发布版本 |
| 只读           | 非 Owner 访问；所有字段灰显；隐藏"存为草稿 / 发布 / 删除 / 共享 / 生成 Token"按钮；"开始对话"按钮仍可用                           |
| 提交中         | "存为草稿"或"发布"按钮 loading；字段只读                                                                                          |
| 提交失败       | 错误条置顶；保留用户输入                                                                                                          |
| 提交成功       | 顶部短暂提示"已存为草稿"或"已发布为 `v<N>`"；发布成功后默认保留当前区段（用户可在顶部 toast 中点"查看版本"跳 UI-008）             |

### 4.3 区段表单字段

#### 4.3.1 基础信息（REQ-003）

## 5. UI-005 · Agent 对话页

**用途**：与某个 Agent 进行多轮对话；支持文本 / 图片 / 语音 / 文档输入（REQ-016）。

### 5.1 布局

- 顶：Agent 头像 + 名称 + 当前模型徽标 + "调试"按钮（跳 UI-007，仅 Owner 可见）+ "返回"按钮（回 UI-004 或 UI-003，看进入路径）
- 左：**历史会话侧栏（OQ-014 closed 2026-05-08）**——可折叠（默认展开）；侧栏顶部为"+ 新建会话"按钮；下方按时间倒序列出该 Agent 下当前用户参与的会话条目（标题 = 首条用户消息前 30 字 + 时间）；点击会话切换主区上下文；折叠后只剩窄竖条与展开把手。本侧栏与 §0.2 左侧主导航 nav **同屏共存**，主导航 nav 在该页面默认收缩为"图标条"以让出宽度。
- 中：消息流区域（按时间顺序，区分用户消息 / Agent 消息 / 工具调用气泡 / 系统提示）
- 下：复合输入区
  - 文本输入框（多行，Enter 发送，Shift+Enter 换行；macOS Cmd ↔ Windows Ctrl 双端等价，OQ-022 closed）

  - 语音按钮：长按录音 / 点击进入录音态；录音过程中触发自动锁定按 §2.5 在途任务特例处理（OQ-017 closed）

| 工具调用中   | 工具气泡展开，显示"调用 <工具名>，参数 <…>"，结果回填后变 hover 可展开  |
| 录音中       | 输入区切换为波形 + 时长计时；显示"取消 / 完成"两个按钮                  |
| 转写中       | 录音完成后显示"转写中…"，转写完成把文本回填到输入框（用户可编辑后发送） |

| 回放中   | 详情区"回放"标签亮起；模型调用使用样本固定参数（v1 不引入差异比对，留 H4） |
| 出错     | "trace 加载失败"，重试按钮                                                 |

### 7.3 表单字段

| 字段         | 必填           | 校验                      |
| ------------ | -------------- | ------------------------- |
| 评测样本名称 | 保存样本时必填 | 1–50 字符，Owner 维度唯一 |
| 备注         | 否             | ≤ 500 字                  |

### 7.4 操作按钮

| 按钮        | 行为                                                   |
| ----------- | ------------------------------------------------------ |
| 保存为样本  | 弹窗收集名称 / 备注；保存后样本可在评测面板列出        |
| 回放        | 把样本输入再次发往后端；列入新的 trace；不影响原 trace |
| 删除        | 仅 Owner 可删自己保存的样本                            |
| 复制 prompt | 详情区每一步右上角"复制"图标                           |
| 跳到对话    | 详情区"返回该对话" → UI-005                            |

### 7.5 错误提示

| 场景         | 文案                                                          |
| ------------ | ------------------------------------------------------------- |
| 样本同名     | "样本名 <名称> 已存在，请改名"                                |
| 回放被限流   | "回放被速率限制，`<n>` 秒后重试" + 倒计时                     |
| trace 缺片段 | "<段名> 数据缺失（可能由后端旧版本不写）"，hover 显示具体字段 |

### 7.6 空状态

| 场景       | 文案                                              |
| ---------- | ------------------------------------------------- |
| 列表空     | "暂无 trace"                                      |
| 评测样本空 | "还没有评测样本，从 trace 详情『保存为样本』开始" |

### 7.7 加载状态

列表与详情独立骨架。

### 7.8 权限差异

## 10. 跨页面通用规约

### 10.1 错误提示模式（与 EX-001 ~ EX-009 对齐）

| 场景级别 | 形式                | 出现位置        | 自动消失                |
| -------- | ------------------- | --------------- | ----------------------- |
| 全局     | 顶部错误条          | 主窗口顶部      | 不自动消失              |
| 页面     | 页面顶错误条        | 当前页面顶      | 用户手动关 / 修复后消失 |
| 字段     | 字段下红字          | 字段正下方      | 修复后立即消失          |
| 流内     | 消息气泡 / 工具气泡 | UI-005 消息流内 | 不消失（属对话历史）    |
| 临时反馈 | toast               | 主窗口右下      | 5 秒自动消失            |

### 10.2 空状态规约

- 列表空 = **骨架占位 + 一句话原因 + 主动作按钮**，不使用插画（OQ-021 closed 2026-05-08）。
- 筛选无结果 = 文案"在所选条件内没有结果，请[清除筛选]"。
- 数据被删除 / 撤销共享 = 占位"该资源已不可访问，请联系所有者或管理员"。

### 10.3 加载状态规约

- 列表 → 骨架占位；不在列表区出现转圈 spinner（避免视觉跳动）
- 详情区 → 骨架或"…"小 loading
- 按钮提交中 → 按钮内置 loading 圈
- 流式回复 → 见 UI-005 5.7
