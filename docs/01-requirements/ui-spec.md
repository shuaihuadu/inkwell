# 自定义 Agent 功能 · H1 UI 说明

> 本文档把 [requirements.md](./requirements.md) 第 5–9 节的 REQ / NFR / E 落到具体的页面、控件、状态矩阵与文案。**不**写视觉稿、不写 token 化的设计变量；这些归原型评审与后续详细设计阶段。本文以 Ant Design Pro 后台模板为视觉骨架（[D1](./requirements.md#122-依赖)），所有页面默认 PC + 移动端响应式，i18n 走 zh-CN / en-US 双语，可访问性按 WCAG 2.1 AA（NFR-003）。

## 1. AppShell 与左侧导航

- 采用 Ant Design Pro 的 **ProLayout**（侧栏 + 顶栏 + content）。
- 侧栏一级菜单包含但不限于"我的 Agent"——本特性所有页面都从该入口进入。Inkwell 平台其它入口由别处规范决定，本文不约束。
- 顶栏右上：用户头像 + 下拉（昵称 / 切换语言 / 退出登录）。退出登录后路由由平台层接管。
- 未登录用户访问任何 P1 ~ P5 路由时，由路由守卫拦截并跳转平台登录页（[E5](./requirements.md#9-异常场景)；详见 P5）。

## 2. 页面清单

| 编号 | 名称                  | 路由（示意）                        | 主要 REQ                                                                                                |
| ---- | --------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------- |
| P1   | 我的 Agent 列表页     | `/agents`                           | REQ-001、REQ-002、DB-004、AC-001-1 ~ 2 / AC-001-5                                                       |
| P2   | Agent 编辑页          | `/agents/:id/edit` 或 `/agents/new` | REQ-002、REQ-003、REQ-004、REQ-010、REQ-011、REQ-012                                                    |
| P3   | Skill 编辑器（侧抽屉） | `/agents/:id/edit?skill=:skillId`   | REQ-010 全部 AC                                                                                          |
| P4   | 删除确认弹层          | （Modal，无独立路由）               | REQ-001 / DB-004 / AC-001-5                                                                              |
| P5   | 未登录跳转            | （路由守卫，无 UI）                 | NFR-002 / R7                                                                                             |

> 本期**不**为 vNext 的"分享详情""导入导出""知识库挂载"等保留 UI 路由，避免误判 MVP 范围（OQ-019 = pending）。

### 2.1 P1 我的 Agent 列表页

- **布局**：顶部 PageHeader（标题"我的 Agent"+ 右上"新建 Agent"按钮 + 顶部 tab `全部` / `已删除`）+ 卡片网格（Card grid，每卡片显示头像、名称、最近更新时间、`编辑` / `复制` / `删除` 三个动作按钮）。
- **新建 Agent 按钮**：固定在右上角，点击后路由到 `/agents/new`，进入 P2 空白态。
- **`全部` tab**：列出当前用户名下所有未软删 Agent；按"最近更新"倒序；分页 / 滚动加载具体阈值由 H3 决定。
- **`已删除` tab**：列出当前用户名下处于 7 天软删 Hold 期的 Agent（DB-004）；卡片右下角显示"剩余 N 天"；动作按钮替换为 `恢复` / `永久删除`。
- **卡片右上角的 `…` 菜单**：折叠 `编辑` / `复制` / `删除`，方便移动端响应式（< 768 px 时卡片网格变 1 列）。

### 2.2 P2 Agent 编辑页

PC 端为**三栏布局**，移动端按 [4](#4-移动端响应式降级) 节降级。

#### PC 三栏

```
┌─ TopBar：面包屑 我的Agent / 编辑：<名称> ─────────────────┐
│  [保存]  [取消]                                            │
├──────────────────┬─────────────────┬──────────────────────┤
│ 左：基本信息      │ 中：Instructions │ 右：能力配置          │
│ - 名称（≤50 字符）│ - 多行编辑器     │ Tabs：               │
│ - 头像上传        │ - 字符数提示     │  · Skill（REQ-010）  │
│ - 描述（≤200）    │                  │  · Tool（REQ-011）   │
│                  │                  │  · MCP（REQ-012 占位）│
├──────────────────┴─────────────────┴──────────────────────┤
│ 下半屏：试运行（REQ-004）                                   │
│ - 左：聊天气泡区（Inkwell 平台共用聊天组件）                  │
│ - 右：当前生效的 Instructions + Skill + Tool 摘要（只读） │
└───────────────────────────────────────────────────────────┘
```

- **保存按钮**：仅在表单 dirty 时启用；保存成功后 Toast，按钮回 disabled。
- **取消按钮**：dirty 时弹二次确认 `放弃修改？`。
- **试运行**：下半屏始终可见，不通过额外 tab 切换。每次试运行调用 `gpt-4.1` + 已启用 Skill / Tool（AC-010-7、AC-011-4）；调用计费按 NFR-005 计入主桶。
  - **未来实施形态待定**（OQ-030 / 2026-05-06 评审）：P2 试运行下半屏未来可能改为「独立子页 + iframe 接入」（复用平台现有聊天页面），以避免重复实现并获得与平台一致的会话 / 历史能力。本期原型仍以「同页 React 组件」为视觉锐定；最终形态（同页组件 vs iframe）由 H2 架构阶段定。UI 交互模型在两种实现下保持一致。

#### Tab 详细

- **Skill tab**：
  - 列表展示当前用户的 Skill 库（DB-006），每行 Checkbox + Skill 名称 + description 一行截断。
  - 顶部按钮 `新建 Skill`（打开 P3 抽屉）/ `导入 SKILL.md`（系统文件选择器，单选，仅 .md）。
  - 行内悬浮显示 `编辑` / `查看引用 Agent 数`。`编辑` 打开 P3。
  - **不暴露 YAML 语法**（NFR-001、AC-010-2）。
- **Tool tab**：
  - **固定**两行（首批 = T-1 + T-3，AC-011-1）：
    - 联网搜索：复选框 + 一句话描述 + "首次启用须知"标识；勾选触发 E6 弹层。
    - 当前日期：复选框 + 一句话描述；不弹须知。
  - **不显示**未来扩展的 T-4 ~ T-8 占位项；明确区别于 MCP（避免混淆）。
- **MCP tab**：
  - 整个 tab 灰显（`disabled`），居中显示"MCP 服务即将上线 · 敬请期待"图文（Empty + 一句话能力简述：`通过 MCP server 接入更多第三方能力`）。
  - **无任何可点击控件**（AC-012-2）。
  - 鼠标悬浮 tab 标签时显示 Tooltip：`本期未开放，vNext 启用`。

### 2.3 P3 Skill 编辑器

- 形态：从 P2 右侧滑入的 Drawer（PC 宽 720 px；移动端全屏）。
- **结构化表单**（AC-010-2、OQ-027 = A）；**不**直接展示 YAML：
  1. **基本信息分组**
     - `name`（required，1–64 字符，仅小写字母 / 数字 / 连字符；前端正则 `^[a-z0-9](?:[a-z0-9-]*[a-z0-9])?$`，禁止连续连字符；不允许 emoji 或多语言；与 [agentskills.io 规范](https://agentskills.io/specification) 对齐）。
     - `description`（required，1–1024 字符；多行 textarea；用于 Agent 调用时让模型决定是否启用此 Skill）。
     - `license`（optional，下拉选 `MIT` / `Apache-2.0` / `Proprietary` / `自填`，自填走 64 字符内文本）。
     - `compatibility`（optional，多行 textarea，提示文案：`例如：claude-3.5-sonnet, gpt-4.1`）。
     - `metadata`（optional，键值编辑器；每行 `key` 输入框 + `value` 输入框 + 删除按钮，`新增一行` 按钮在底部）。
  2. **正文分组**
     - `body`（required，Markdown 编辑器；建议用 Ant Design + 团队既有 Markdown 组件；提供"预览"切换；不暴露 YAML）。
- **保存**：前端把上述字段序列化成符合规范的 SKILL.md 文本（YAML frontmatter + Markdown body），发往后端。失败时按 E7 显示具体错误。
- **导入 SKILL.md**（来自 P2 Skill tab 的"导入"按钮，进入 P3 后填充表单）：
  - 读取后解析 frontmatter，校验 `name` / `description` 合规；
  - 失败：弹"无法导入：第 N 行解析失败 / `name` 不合规 / `description` 缺失"等具体提示，停留在导入对话框（E7）。
  - 成功且与当前用户已有 Skill 重名：弹 ConfirmModal `已存在同名 Skill：<name>`，二选一：
    - `覆盖现有`（D2）→ 用导入内容直接覆盖 Skill 库中同名条目（保留版本号 +1 留作 Skill 自身演进，与 Agent 引用解耦）；
    - `另存为新名称` → 自动建议 `<name>-2`，用户可改后另存（OQ-028 = D2）。
- **不**支持 ZIP / 多文件 / Git URL（ND-011、ND-012、AC-010-9）；上传按钮文本明示"仅支持单个 .md 文件"。

### 2.4 P4 删除确认弹层

- 触发：P1 卡片或 P2 顶部"删除"按钮。
- Modal 文案：
  - 标题：`删除 <Agent 名称>？`
  - 正文：`将进入"已删除"列表保留 7 天，期间可恢复；7 天后将永久删除。`（DB-004）
  - 按钮：`取消` / `删除`（type=danger，二次焦点防误触）。
- 删除成功后跳回 P1，并 Toast `已移到"已删除"，7 天内可恢复`。

### 2.5 P5 未登录跳转

- 路由守卫：进入任意 `/agents*` 路由前调用平台登录态接口（D2）；
  - 已登录：放行；
  - 未登录：`Navigate to /login?return=/agents...`（具体登录路由由平台决定，本特性不写死）；
  - 接口异常：跳转平台 `/login` 不带 return，避免泄露当前页面（防止 R7 退化时把本特性陷在 401 循环）。
- 本页**无独立 UI**；本节存在仅为合规化路由约束。

## 3. 状态矩阵

> 每页都需覆盖：默认 / 加载中 / 空 / 错误 / 网络断开 / 未登录 / 调用重试中 / 重试 3 次仍失败。下表只给出"非默认"的 8 类状态。

### 3.1 P1 列表页

| 状态           | UI 表现                                                                 | 触发                            |
| -------------- | ----------------------------------------------------------------------- | ------------------------------- |
| 加载中         | Card grid 用 Skeleton 卡片占位（≥ 6 个），不显示空状态                    | 列表接口请求中                  |
| 空             | Empty 组件 + 文案 `还没有 Agent，点击右上角"新建"开始`，下方一个 CTA 按钮 | 列表接口返回长度为 0           |
| 错误           | Empty 组件（image=error）+ 文案 `加载失败` + `重试` 按钮                 | 列表接口非 401 / 非 0 长度失败 |
| 网络断开       | 顶部黄色 Alert：`网络已断开，正在重试…`；列表保留上次缓存                | 浏览器 `offline` 事件 / 5s 超时 |
| 未登录         | 不渲染列表，立刻跳 P5                                                    | 路由守卫触发                    |
| `已删除` 空    | Empty 组件 + 文案 `已删除列表为空`，无 CTA                                | 软删除列表为 0 长度            |

### 3.2 P2 编辑页

| 状态                                       | UI 表现                                                                                                     | 触发                                                  |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| 加载中（从 :id 拉取已有 Agent）              | 全屏 Spin                                                                                                    | `/agents/:id/edit` 首次进入                            |
| 表单 dirty + 离开                           | 浏览器 `beforeunload` 提示 + 路由切换前 ConfirmModal `离开会丢失未保存修改？`                              | 用户切路由 / 关页面                                    |
| 试运行调用中                                 | 聊天气泡区显示打字省略号；右侧"摘要面板"灰显                                                                 | 试运行接口 in-flight                                  |
| 试运行失败 / 第 1 次重试中                   | 聊天气泡显示 `网络抖动，正在自动重试 1/3…`（E1）                                                              | 试运行接口失败，重试自动开启                          |
| 试运行 3 次仍失败                            | 聊天气泡显示系统消息 `出错了，请稍后重试`，下方 `重试` 按钮（E1）                                            | 重试上限触发                                          |
| 试运行被网络断开打断                         | 顶部 Alert（黄色）`网络已断开，等待恢复后将继续保存最近一次保存于 <时间>`（E5）                              | 浏览器 `offline`                                      |
| Skill / Tool / 模型调用产生计费扣减           | 右下角小型计费角标 `本次试运行消耗 ~N tokens`（NFR-005）                                                     | 单次试运行返回                                        |
| 首次启用 T-1 联网搜索                       | Modal `用户须知：联网搜索会把您的对话发送给搜索服务方` + `我知道了`（E6）                                     | 用户首次勾选 T-1                                       |

### 3.3 P3 Skill 编辑器

| 状态           | UI 表现                                                                          | 触发                                  |
| -------------- | -------------------------------------------------------------------------------- | ------------------------------------- |
| 校验失败       | 字段下方红色提示 `name 必填且需符合 ^[a-z0-9-]+ 规则`                            | 失焦 / 提交                           |
| 保存中         | 抽屉底部按钮 disabled + Spin                                                     | 保存接口 in-flight                    |
| 导入解析失败   | 上传组件下方 Alert（红色）`第 3 行：name 包含不允许的字符 '_'`（E7）              | 后端解析返回错误位置                  |
| 重名冲突       | ConfirmModal `已存在同名 Skill：<name>`，按钮 `覆盖现有` / `另存为新名称`（D2）   | 后端返回 409 同名                    |

### 3.4 P4 删除确认 / 恢复

| 状态     | UI 表现                                                          |
| -------- | ---------------------------------------------------------------- |
| 删除中   | 按钮 Spin                                                        |
| 删除失败 | 抽屉内 Alert（红色）`删除失败：<原因>` + `重试` 按钮             |
| 恢复成功 | Toast `已恢复 <Agent 名称>`                                      |
| 永久删除 | 二次 Modal `永久删除后无法恢复，确认？`（红色 type=danger）     |

## 4. 移动端响应式降级

- **断点**：< 768 px 进入移动端布局（D1，OQ-015 = B）。
- **P1 列表页**：卡片网格变 1 列；右上"新建 Agent" 收为浮动 FAB（右下角）；顶部 tab 不变。
- **P2 编辑页**：三栏 + 下半屏 → **纵向 Tab 栈**：`基本信息` / `Instructions` / `Skill` / `Tool` / `MCP` / `试运行`，每个 tab 一屏全宽。Tab 顺序固定；试运行始终在最后一个 tab，避免误触。
- **P3 Skill 编辑器**：Drawer 全屏；frontmatter 字段折叠为可展开的 Collapse Panel，默认仅展开"基本信息"。
- **P4 Modal**：底部弹层（drawer-from-bottom）。
- **P5 路由守卫**：行为不变。

## 5. 错误文案标准（与 [requirements.md](./requirements.md) 第 9 节对齐）

| 异常 | 用户能看到的文案                                                                                                        | 位置                       |
| ---- | ----------------------------------------------------------------------------------------------------------------------- | -------------------------- |
| E1   | `网络抖动，正在自动重试 N/3…` → 失败时 `出错了，请稍后重试`                                                              | 试运行聊天气泡（系统消息） |
| E5   | `网络已断开，等待恢复后将继续保存最近一次保存于 <HH:mm:ss>`                                                              | 顶部 Alert（黄色）         |
| E6   | `用户须知：本工具会将对话内容发送给 [服务方]，是否继续？`                                                                | Modal                      |
| E7   | `无法导入：第 N 行：<具体错误>`（如 `name 包含不允许的字符 '_'` / `description 不能为空`）                              | 导入抽屉内 Alert（红色）   |

> 文案需提供 zh-CN / en-US 双语对照（NFR-003）。

## 6. i18n 与无障碍（NFR-003）

- 所有可见文本走 i18n key，禁止在组件里硬编码字面量。
- 数字 / 日期 / 时间格式按用户语言（zh-CN：`YYYY 年 MM 月 DD 日 HH:mm`；en-US：`MMM DD, YYYY HH:mm`）。
- WCAG 2.1 AA：颜色对比度 ≥ 4.5:1（正文）/ ≥ 3:1（大文字）；所有交互元素键盘可达；Modal 必须焦点陷阱；表单错误提示必须 `aria-describedby` 关联到字段；试运行聊天气泡的"系统消息"必须 `role="status"`。
- 移动端的 FAB 大小 ≥ 44 × 44 px，符合可点击目标最小尺寸。

## 7. 与原型 / 详细设计的边界

- **本文档不涵盖**：视觉规范（色板 / 字号 / 间距 token）、动效曲线、URL 参数细节、后端 API schema、试运行的 token 流式协议、模型路由策略——属于原型评审 + H3。
- **后续阶段需补**：原型截图（按页面 P1 ~ P5）、UI 评审记录（`docs/07-reviews/YYYY-MM-DD-h1-ui.md`），评审通过后 [requirements.md](./requirements.md) 与本文 frontmatter `status` 同步切到 `reviewed`。
