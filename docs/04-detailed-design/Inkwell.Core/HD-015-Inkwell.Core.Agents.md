>
> **范围核实结论（非臆造，逐条附证据）**：
>
> - **REQ-002（Agent 列表与基础管理）+ REQ-003（基础属性）+ REQ-004（Instructions）全部在范围内**——[requirements.md §5.1](../../01-requirements/requirements.md) 字面"列表 / 新建 / 编辑 / 删除 / 共享"+"名称（1–50 字）/ 头像 / 描述（≤500字）"+"多行文本，无字数硬上限"；[§11 验收标准](../../01-requirements/requirements.md) REQ-002/003/004 三条逐一对应本 HD 的 CRUD + 共享 + 基础字段设计。
> - **REQ-005（模型选择）+ REQ-006（模型参数配置）仅"Agent 侧持久化引用"部分在范围内**——`AgentDefinition` 持有 `ModelId`（不透明字符串引用）+ `ModelParameters`（复用 [HD-006 `AgentModelParameters`](../Inkwell.Abstractions/HD-006-Inkwell.Abstractions-agent-runtime-port.md)），仅作为**数据存储**；模型注册表本身（后端配置文件声明的模型清单、厂商路由、可用性校验，[AC-020](../../01-requirements/acceptance-criteria.md) ~ [AC-022](../../01-requirements/acceptance-criteria.md)）不在本 HD——归 `Inkwell.Core.Models`（[AGENTS.md §3.1](../../../AGENTS.md) 已锁定的独立业务命名空间，未起草）。本 HD **不**校验 `ModelId` 是否为已注册的合法模型，该校验职责留给 `Inkwell.Core.Models`（或 `Inkwell.WebApi` 组合两者时）。
> - **REQ-007（Function Calling / 工具调用）仅"Agent 侧工具绑定"部分在范围内**——`AgentDefinition` 持有 `ToolBindings`（工具 ID + 静态参数的引用集合），仅作为**数据存储**；工具注册表本身（哪些工具存在、参数 schema、运行期调用代理，[AC-025](../../01-requirements/acceptance-criteria.md) ~ [AC-028](../../01-requirements/acceptance-criteria.md)）不在本 HD——归 `Inkwell.Core.Tools`（未起草）。本 HD 不校验 `ToolBinding.ToolId` 是否存在、不校验必填参数是否齐全（[AC-028](../../01-requirements/acceptance-criteria.md) 的"缺少必填参数"前端 / 后端二次校验归 `Inkwell.Core.Tools` 或 `Inkwell.WebApi` 组合层）。
> - **REQ-008（Agent Skills）仅"Agent 侧 Skill 挂载"部分在范围内**——`AgentDefinition` 持有 `SkillBindings`（Skill ID 引用集合），仅作为**数据存储**；Skill 静态加载（`SKILL.md` 解析、`scripts/` 拒收、Discovery → Activation → Execution）不在本 HD——归 `Inkwell.Core.Skills`（未起草，[ADR-010](../../03-architecture/adr/ADR-010-skill-loading-static-only-v1.md)）。
> - **REQ-015（版本管理）仅"版本号递增触点"在范围内**——`AgentDefinition.CurrentVersion`（`int`）在每次 `UpdateAgentAsync` 成功后 `+1`，满足"每次保存自动生成版本号"的最小语义；版本快照存储、diff、回滚（[AC-056](../../01-requirements/acceptance-criteria.md) ~ [AC-059](../../01-requirements/acceptance-criteria.md)）**不**在本 HD——归 `Inkwell.Core.Versioning`（未起草）。**证据**：[file-structure.md](../file-structure.md) 现有模板已把 `Persistence/Versioning/AgentVersion.cs` + `IAgentVersionRepository.cs` 独立列在 `Versioning/` 子目录（非 `Agents/` 子目录），确认版本快照的物理归属早已被规划为独立模块，本 HD 不越权创建该文件。

## 1. 模块概述

### 1.1 职责

`Inkwell.Core.Agents` 承担：

- Agent 定义的 CRUD（创建 / 查询 / 更新 / 删除）+ 列表（"我的" / "团队共享" 两档，[REQ-002](../../01-requirements/requirements.md)）
- Agent 基础属性（名称 / 头像 / 描述，[REQ-003](../../01-requirements/requirements.md)）与 Instructions（[REQ-004](../../01-requirements/requirements.md)）的持久化
- Agent 侧的模型选择引用 + 模型参数（[REQ-005](../../01-requirements/requirements.md) / [REQ-006](../../01-requirements/requirements.md)）、工具绑定（[REQ-007](../../01-requirements/requirements.md)）、Skill 挂载（[REQ-008](../../01-requirements/requirements.md)）**数据存储**（不含各自的注册表 / 执行逻辑，详见顶部 callout）
- Agent 共享 / 撤销共享（Owner 自助 + Admin 强制撤销，[REQ-002](../../01-requirements/requirements.md) + [REQ-017](../../01-requirements/requirements.md) 子能力）+ "复制为我的 Agent"（[requirements.md §7](../../01-requirements/requirements.md)）
- 版本号递增触点（`CurrentVersion` 字段，[REQ-015](../../01-requirements/requirements.md) 最小语义，详见顶部 callout）
- `AgentDefinition` → [HD-006 `AgentRunRequest`](../Inkwell.Abstractions/HD-006-Inkwell.Abstractions-agent-runtime-port.md) 的翻译与调用衔接（`IAgentInvocationService`）
- Agent CRUD / 共享 / 调用的审计写入触点（[NFR-004](../../01-requirements/requirements.md)，经 [HD-007 `IAuditLogger`](../Inkwell.Abstractions/HD-007-Inkwell.Abstractions-audit-logger-port.md)）

`IAgentService` / `IAgentInvocationService` 是本模块**业务对外接口**，落在 `Inkwell.Abstractions/Agents/`（[HD-001 §5.1](../Inkwell.Abstractions/HD-001-Inkwell.Abstractions-foundation.md#51-命名) 命名约定 "`I<Module>Service`（业务端口）"，判断依据见 [§1.4](#14-iagentservice--iagentinvocationservice-端口位置判断作者判断非-owner-拍板)）。

### 1.4 `IAgentService` / `IAgentInvocationService` 端口位置判断（作者判断，非 Owner 拍板）

同 [HD-014 §1.4](HD-014-Inkwell.Core.Auth.md#14-iauthservice-端口位置判断作者判断非-owner-拍板) 的推理路径：两者目前均只由 `Inkwell.WebApi`（未起草）调用（CRUD / 共享 / 调用三类用例均是 HTTP 端点直接消费）。按"业务对外接口"这一既定内容类别（[HD-001 §5.1](../Inkwell.Abstractions/HD-001-Inkwell.Abstractions-foundation.md#51-命名)），沿用 [HD-006](../Inkwell.Abstractions/HD-006-Inkwell.Abstractions-agent-runtime-port.md) / [HD-007](../Inkwell.Abstractions/HD-007-Inkwell.Abstractions-audit-logger-port.md) / [HD-014](HD-014-Inkwell.Core.Auth.md) 已确立的先例（单实现仍落端口层，便于 WebApi 单元测试 mock），本 HD 同样把两个接口放入 `Inkwell.Abstractions/Agents/`。

`IAgentInvocationService` 单独拆分（不合并进 `IAgentService`）的理由：`IAgentService` 是 CRUD / 共享类"管理面"操作，`IAgentInvocationService` 是"数据面"（每次对话都会调用，性能路径与 CRUD 完全不同，[HD-006 §7.1](../Inkwell.Abstractions/HD-006-Inkwell.Abstractions-agent-runtime-port.md#71-性能预算) 已单独定义 `IAgentRuntime` 的性能预算），拆分后 `Inkwell.WebApi` 的对话端点只需依赖 `IAgentInvocationService`，不需要引入 CRUD 相关依赖面，符合接口隔离原则。
